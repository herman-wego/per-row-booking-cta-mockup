What Changed vs. the Current Page
The only addition is a "Check current prices" button on each row. Everything else — the Emirates logo, flight number, codeshare info, origin/destination with scheduled + actual times, terminal, flight path line, duration, status badge, share icon — is preserved exactly as it appears on the live page. The button links to Wego flight search with the row's origin and destination pre-filled.

Design Decisions
Preserves existing UI: Every element from the current flight status row is kept — Emirates logo, flight number, codeshare links, origin/dest with scheduled + actual times (strikethrough + red/green), terminal, flight path dots, duration, status badge, share icon. Nothing removed.

Soft CTA copy (per Faical's feedback): "Check current prices" feels informational — it matches the user's status-checking mindset. The link behind each button passes the correct origin → destination to Wego's flight search.

Desktop: Outlined button sits inline at the end of the row. Subtle enough to not overpower the status data, but always visible. Fills green on hover.

Mobile: Full-width outlined button below each row's duration/status line. Large enough touch target. Fills green on tap.
